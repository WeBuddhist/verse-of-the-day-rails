### 84000 availability — checked, Toh 12/13/16/53/60 can't be imported yet

Checked `84000/data-tei` (their public GitHub repo of published translations, ~396 texts) directly: **Toh 12, 13, 16, 53, and 60 are not among them.** 84000 simply hasn't published an English translation of these five sūtras yet (confirmed by extracting every Toh number present in the repo's filenames — the sequence runs …11, 19, 21… skipping 12–18 entirely, and similarly skips 53 and 60).

The raw Tibetan **is** available from OpenPecha's Degé Kangyur digitization (`P000001`) for all five — checked via its `index.yml`, spans confirmed present for T12, T13, T16, T53, T60. But this vault's own rail-building approach depends on a paired reference translation to ground meaning (`CLAUDE.md`: "no commentaries, so rails are translation-grounded"); importing Tibetan-only would mean building rails for these five with no translation aid at all, unlike every other Kangyur text in the vault. Also worth noting for scope: T12's span is enormous (~907,000 characters across two volumes) and T60's is large too (~520,000 characters) — both would be bigger imports than anything in the vault so far, for a single quotable lead each.

**DECISION (Evan, 2026-07-13): option (b) — go ahead and use these without a paired reference translation.** We accept that these five will have to be translated directly from the Tibetan with no 84000 reference to ground meaning, and that this is OK for these texts. Practical consequences before any of the five actually ships:
- **Import is still required** — no `bo-toh<N>.md` exists for 12/13/16/53/60 yet; import the raw Tibetan from OpenPecha's Degé Kangyur (`P000001`) and register each in `vault-annex.md`'s corpus table + §7 licensing per `CLAUDE.md`. (Scope reminder: T12 ≈ 907k chars, T60 ≈ 520k chars — large imports for a single lead each; 13/16/53 are smaller.)
- **`verse-rail` currently assumes a paired reference translation** (the vault's "translation-grounded" premise). Building a rail for these five means running *without* that reference — a genuine carve-out from that premise, not just a missing file. **This needs a matching note in `verse-rail`/`CLAUDE.md` before it's exercised — flagged for Evan, not edited unilaterally here.** All the hard gates (buddhavacana, verbatim-from-source, in-scope discourse/verse) still apply exactly as before; only the reference-translation aid is being waived.
- Option (a) skip-and-wait and (c) find-another-translation are now off the table for these five unless Evan reverses this.
